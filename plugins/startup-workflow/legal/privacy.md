# Privacy Policy

The Startup Workflow plugin operates on content that the user provides in Codex and on files inside the current workspace when the workflow explicitly chooses to read them.

The plugin itself does not define external data collection, tracking, or analytics.

What the plugin may access:

- prompts and instructions entered by the user
- workflow files written inside the current workspace
- local project files when the workflow or the user explicitly opts into using them

What the plugin does not intentionally introduce:

- background telemetry
- external account linkage
- independent storage outside the workspace and the host Codex environment

Users are responsible for reviewing any underlying Codex, model, connector, or platform policies that apply to their environment.
